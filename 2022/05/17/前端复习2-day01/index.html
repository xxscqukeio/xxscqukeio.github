前端基础-js1

2.1 let const var 相关 声明与作用域 *3

var–ES5 变量声明方式

1.变量未赋值时使用 不会报错 值为undefined

2.作用域 为方法作用域 只要定义了 整个方法内变量都可用

3.可以重复声明变量

let–ES6 变量声明方式

1.let 声明前使用会报错

2.作用域为 块作用域 通常let比var 范围要小

3.let禁止重复声明 否则会报错

const–ES6 变量声明方式

变量指向的内存地址不能改动

1.const常量声明方式；声明方式必须初始化 且后面出现的代码中不能修改

2.const实际上保证的 并不是变量值不得改动 而是变量指向的内存地址不能改动

2.2 js数据类型，区别

基本数据类型：*7 3+2+2

number string boolean

null undefined

symbol bigint

引用数据类型：*2

object function

object: 普通对象 数组对象 正则对象 日期对象 math数学函数对象

存储方式：

基本数据类型：占用空间小 大小固定 被频繁使用

基本数据类型 存储在 栈中的 简单数据段

栈是 存储基本类型值 和 执行代码 的空间

引用数据类型：占据空间大 大小不固定

存储在堆内存中 栈中存储指针 指针指向堆中该实体的起始地址

解释器寻值时 获取栈中的地址 然后根据地址从堆中获得实体

区别：*4 大 无 稳 动

1.堆比栈 空间大 栈比堆运行速度快

2.堆内存是 无序存储 可以根据引用直接获取。

3.基本数据类型比较稳定 相对占用内存小

4.引用数据类型 动态的 无限

2.3 Object.assign的理解

Object.assign可以实现对象的合并 *3

1.将source中 可枚举的属性 复制到 target 重名则覆盖

2.后续的source会覆盖前面source的同名属性

3.方法复制的是属性值 若属性值为引用数据类型—-引用地址的复制 引发引用共享问题

2.4 constructor的理解 ？

每个函数—-对应一个prototype（原型）对象

属性是一个指针，指向一个对象

构造函数的prototype设置为等于一个以对象字面量形式创建的新对象时，constructor属性不再指向该构造函数

2.5 map 和 forEach 的区别

4*相同点

1.都是循环遍历数组中的每一项

2.执行匿名函数都支持3个参数 （item ，index，arr）

3.匿名函数中的this都指向window

4.只能遍历数组

2*不同点

map()会分配内存空间 存储新数组并返回 forEach()不会返回数据

forEach()允许callback更改原始数组元素 map()则是返回新的数组

2.6 for of 可以遍历哪些对象

for of 只限于迭代器iterator*6 2A 2S T M

普通对象会报错

可迭代对象 Array、Map、Set、String、TypedArray、arguments对象等

2.7 js静态类型检查

js是动态类型语言

静态类型语言：类型检查发生在编译阶段 除非修复错误 否则一直编译失败

动态类型语言：只有在程序运行了一次的时候错误才会被发现 运行时包含了会 在运行时阻止脚本运行的错误类型 也可以通过编译

js静态类型检查的方法

Flow开源的静态类型检查库 逐渐地向JavaScript代码中添加类型

Typescript编译为JavaScript的超集 新的静态类型语言

使用静态类型的优势*3

尽早发现bug和错误

减少复杂的错误处理

数据和行为分离

劣势 长 时

代码冗长

花时间掌握类型

  2. 使用indexOf检索数组时 使用**”===”匹配 会检查数据类型**

语法：

第一出现的索引 没有找到返回-1

2.9 iframe有什么优点、缺点

优点*4 原 多 重 慢

  1. 原封不动的把嵌入网页展示出来
  2. 多个网页引用iframe 只需修改iframe内容 就可以修改每一个
  3. 网页同一风格 可以 写成一个页面 用iframe来嵌套 代码的可重用
  4. 加载缓慢的第三方内容（图标、广告）可以由iframe来解决

缺点：*6 阻 并 多 搜 兼 服

  1. iframe阻塞主页面的onload事件
  2. iframe和主页面共享连接池 影响页面并行加载 产生很多页面 不易管理
  3. iframe框架多–滚动条–影响注意力–用户体验差
  4. 代码复杂–不利于seo
  5. 移动端无法完全显示矿建 设备兼容性差
  6. 增加服务器http请求 对大网站不可取
