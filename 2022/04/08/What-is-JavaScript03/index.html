语言基础 （语法+声明）

1.语法

ECMAScript的语法 大程度借鉴了C、类C语言（Java和Perl）

区分大小写

无论变量、函数名、操作符都区分大小写

typeof是个关键字 Typeof是一个完全有效的函数名

允许非英文字母，但不推荐

标识符

首字符：字母、_、 $ 必须非数字。

剩下的字符可以加入数字

标识符 驼峰大小写形式

firstSecond、myCar、doSomethingImportant

常见关键字、保留字、true、false、null不能做

'$' 和 '_' 也可以用于变量命名。

正常的符号，就跟字母一样，没有任何特殊的含义。

注释

// 单行注释

或/*多行注释*/

Ctrl+/ 热键进行单行注释，诸如 Ctrl+Shift+/ 的热键可以进行多行注释

不支持注释嵌套！

不要在 /*...*/ 内嵌套另一个 /*...*/。

现代模式，”use strict”

"use strict" 来明确地激活

ES5 规范增加了新的语言特性并且修改了一些已经存在的特性。

确保 “use strict” 出现在最顶部

请确保 "use strict" 出现在脚本的最顶部，否则严格模式可能无法启用。

一旦进入了严格模式，就没有回头路了。

语句

分号非必须 但是推荐

大部分时候可以省略分号，但是最好不要省略分号，尤其对新手来说。

代码块{}可以使内容更清晰

2.变量

松散类型 变量 可以保存任何类型的数据

3个关键字 可以声明:

var (所有)

const let (es6之后)

不推荐改变 变量保存值 类型

var 声明作用域

局部作用域定义 全局变量 很难维护 且不能在严格模式下使用

定义并初始化多个变量

为了更好的可读性，请一行只声明一个变量。

var 声明提升

使用var会将变量自动升到函数 作用域顶部

提升(hoist) 把所有变量声明 提升到 函数作用域的顶部

反复声明同一个变量 以最后声明的值为准

let 声明 4个区别

let 声明 范围 ： 块作用域

var 声明 范围：函数作用域

let声明的age 作用域仅限于块内部

let 不允许在同一个作用域出现冗余声明 (重复声明) SyntaxError 标识符age

let 和 var 混用 依然会垂涎冗余声明报错

let变量不会在作用域中被 提升

全局声明

let在全局作用域中声明 不会变成window对象 属性

window.age // undefined

页面中 不要重复声明 同一个变量

对同一个变量应该只声明一次，之后在不使用 let 的情况下对其进行引用

条件声明

不能使用let 进行条件声明

for循环中的let声明

var 在for循环中定义的迭代变量 会渗透到 循环体外部

let 仅限于for循环内部 循环外的i未定义

const 声明

声明变量时 需要 同时初始化

也不能重复声明、并且作用域也是块

当程序员能确定这个变量永远不会改变的时候，就可以使用 const 来确保这种行为

const声明指向变量引用的是 一个对象

可以修改对象内部属性 同时也不违背const限制

大写形式的常数

3.声明风格

  1. 不使用var

  2. 优先使用const 再使用let

  3. 声明一个常数（不变）变量，可以使用 const 而非 let：

  4. const — 类似于 let，但是变量的值无法被修改。
