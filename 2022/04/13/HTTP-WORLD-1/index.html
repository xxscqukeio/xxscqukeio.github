与HTTP相关的各种概念

HTTP 相关的各种应用 互联网、浏览器、Web 服务器

网络世界

Internet，里面存储着无穷无尽的信息资源

我们通常所说的“上网”实际上访问的只是互联网的一个子集“万维网”（World Wide Web） 互联网 90% 以上的部分都被万维网

n zz它基于 HTTP 协议，传输 HTML 等超文本资源，能力也就被限制在 HTTP 协议之内

常用的电子邮件、BT 和 Magnet 点对点下载、FTP 文件下载、SSH 安全登录、各种即时通信服务

HTTP 协议非常灵活、易于扩展，而且“超文本”的表述能力很强

“网页应用”——例如“微信网页版”“邮箱网页版

浏览器

浏览器本质上是一个 HTTP 协议中的请求方

使用 HTTP 协议获取网络上的各种资源

HTML 排版引擎用来展示页面

JavaScript 引擎用来实现动态化效果

甚至还有开发者工具用来调试网页，以及五花八门的各种插件和扩展。

User Agent 即“用户代理”—-客户端

作为访问者的“代理”来发起 HTTP 请求。

Web 服务器

服务器，Web Server—-应答方（响应方）

响应请求的主体

“Web 服务器”时有两个层面的含义：硬件和软件

物理形式或“云”形式的机器 反向代理、负载均衡等技术组成的庞大集群

虚拟的

软件含义的 Web 服务器

就是提供 Web 服务的应用程序，通常会运行在硬件含义的服务器上。

强大的硬件能力响应海量的客户端 HTTP 请求

Apache

Nginx

CDN

CDN，全称是“Content Delivery Network

HTTP 协议里的缓存和代理技术，代替源站响应客户端的请求

可以缓存源站的数据，让浏览器的请求不用“千里迢迢”地到达源站服务器，直接在“半路”就可以获取响应。

CDN 的调度算法很优秀，更可以找到离用户最近的节点，大幅度缩短响应时间。

CDN 也是现在互联网中的一项重要基础设施，除了基本的网络加速外，还提供负载均衡、安全防护、边缘计算、跨运营商网络等功能，

成倍地“放大”源站服务器的服务能力

爬虫

浏览器，它是一种用户代理，代替我们访问互联网。

HTTP 协议没有规定用户代理后面必须是“真正的人类”—-机器人—-爬虫”（Crawler）

可以自动访问 Web 资源的应用程序

互联网上至少有 50% 的流量都是由爬虫产生的

抓取网页存入庞大的数据库，再建立关键字索引

君子协定”robots.txt

基本技术都是两个 HTTP、HTML。

HTML/WebService/WAF

HTML 是 HTTP 协议传输的主要内容之一

用标签 定义资源和排版布局

Web Service

W3C 定义的应用服务开发规范

一个基于 Web（HTTP）的服务架构技术

Web Service 架构里服务器和客户端可以采用不同的操作系统或编程语言开发

服务器端用 Linux+Java，客户端用 Windows+C#，

WAF

“网络应用防火墙” 位于 Web 服务器之前

专门检测 HTTP 流量，是防护 Web 应用的安全技术

阻止SQL 注入、跨站脚本等攻击，目前应用较多的一个开源项目是 ModSecurity，它能够完全集成进 Apache 或 Nginx。

小结

互联网上绝大部分资源都使用 HTTP 协议传输；

浏览器是 HTTP 协议里的请求方，即 User Agent；

服务器是 HTTP 协议里的应答方，常用的有 Apache 和 Nginx；

CDN 位于浏览器和服务器之间，主要起到缓存加速的作用；

爬虫是另一类 User Agent，是自动访问网络资源的程序。

你觉得 CDN 在对待浏览器和爬虫时会有差异吗？为什么？

CDN 应当是不区分的

因为爬虫本身也是对 Web 资源的访问，且对于爬虫识别并不是 100% 准确的

因此 CDN 只会去计算实际使用了多少资源而不管其中多少来自爬虫；

你怎么理解 WebService 与 Web Server 这两个非常相似的词?

Web Service 是网络服务实体

Web Server 是网络服务器，后者的存在是为了承载前者。

暗网理解成一种局域网
