HTML中的JavaScript

<script>元素

8个属性

async：可选 立即开始下载脚本 不能阻止其他页面动作（下载资源或等待其他脚本加载） 只对外部脚本文件有效

charset：可选 指定代码字符集 多数浏览器不在乎

crossorigin：可选 跨域 默认不适用CORS

differ、integrity 可选

language 废弃 被 type 代替

src 可选 要执行的代码的外部文件

如果设置了 src 特性，script 标签内容将会被忽略。

一个单独的 <script> 标签不能同时有 src 特性和内部包裹的代码

需要写成2个<script> 标签

type (MIME脚本语言的内容类型) 可选 按例都是”text/javascript”

当type=module会被当成ES6 代码会出现import、export

2种使用方法

嵌入

行内代码 不能出现</ script> 会导致报错

若要输入可以 使用 转义字符 “\”

使用src调用外部文件

浏览器会按照<script> 在页面中出现的顺序依次解释他们

只有最简单的脚本才嵌入到 HTML 中。更复杂的脚本存放在单独的文件中。

使用独立文件的好处是浏览器会下载它，然后将它保存到浏览器的 缓存 中。

外部文件优点

  1. 可维护性
  2. 缓存
  3. 适应未来

留意 SPDY/HTTP2 新的浏览器支持

能批量 逐个放到浏览器缓存中

标签位置

过去

把外部CSS和JavaScript文件都集中放在一起 <head> 标签内

但是如果 需要浏览器解释完所有JavaScript代买后 才能渲染页面

页面是在<body>的启示标签时开始渲染 导致页面渲染的明显延迟

现在

将JavaScript引用放在<body> 元素中页面内容的后面

页面会在处理JavaScript代码前完全渲染页面 用户会感觉到加载更快了

推迟执行脚本

<script defer src=" "> </ script>

HTML5 会忽略行内脚本defer

通常将推迟执行的脚本放在页面底部

IE已死 应该废弃

异步执行脚本

<script async src=" "> </ script>

不必等脚本下载和执行后在加载页面

支持但不推荐使用

动态加载脚本

通过DOM API 添加<script>元素

XHTML extensible 超文本标记语言

废弃 担忧遗留

转义字符 > < 等

或者使用 CDATA

文档模式

doctype切换文档模式

混杂模式 省略doctype声明为开关

标准模式 准标准模式 很接近（除混杂模式以外的都可以叫标准）

noscript 元素

用于不支持或禁用JavaScript的浏览器

如果浏览器支持JavaScript则永远看不到noscript里面的内容

总结

  1. 引入外部JavaScript文件 设置src :包含文件的URL 可以同一个服务器上 也可以完全不同的域
  2. 按照<script>在网页上出现的次序一次解释 不使用defer、async会严格按次序解释
  3. 通常将<script>元素放在页面末尾 介于主内容和</body>之间
  4. defer可以推迟 但将死 async能够不等待 但不能保证次序
  5. <noscript>指定不支持脚本时显示的内容
