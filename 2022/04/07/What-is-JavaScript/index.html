1.1 JavaScript简介

创建目的：“网页更生动”

脚本语言 放在HTML网页中 会在页面加载时自动执行

脚本语言 纯文本形式 提供与执行 不需要特殊的准备、编译即可运行

这方面区别于Java、c等

livescript–JavaScript 为了蹭热度

随着发展 有了语言规范ECMAScript

执行环境 不限于浏览器、服务端、搭载JavaScript引擎就能执行

浏览器都会嵌入JavaScript引擎（JavaScript虚拟机）

V8 –Chrome Opera Edge

Spidermonkey –Firefox

IE如今已经彻底沦为过去式

引擎是如何工作的？

3步：解析、编译、执行

  1. 引擎（如果是浏览器，则引擎被嵌入在其中）读取（“解析”）脚本。
  2. 将脚本转化（“编译”）为机器语言。
  3. 机器代码快速地执行。

浏览器中的JavaScript （5 ）

1.网页中添加新的HTML 修改内容和样式

2.响应用户行为、响应鼠标的点击、指针的移动、按键的按动 （各种事件的响应）

3.向远程服务器发送请求，Ajax、COMET 下载和上传文件

4.获取、设置cookie 向访问者提出问题、发送消息

5.记住客户端的数据（“本地存储”）

浏览器中的JavaScript不能

1.不能读、写、复制、执行硬盘上的任意文件。 没有直接访问操作系统的功能

与相机、麦克风等设备交互 需要许可 理论上用户信息是安全的

2.不同标签页、窗口互相不了解，

2个标签页打开的不是同一个网站，不能相互通信。（域名、协议、端头都需要相同）“同源策略” 为了用户信息安全 防止被窃取信息

3.JavaScript可以轻松通过互联网与页面所在服务器通信

但是从其他网站、域 接受数据能力被削弱 需要来自远程服务器的明确协议

（http header中） 也还是为了用户信息安全

在浏览器外 不存在此类限制 浏览器还有扩展权限的插件、扩展

JavaScript 3个与众不同

与HTML、CSS的 完全集成

简单

所有主流浏览器支持 并默认开启

还以用于服务器、移动端等

JavaScript语言

现在是一个上层语言

其他语言编译成JavaScript

Typescript 严格的数据类型

kotlin Node 中使用

总结

最初专门为浏览器 现在用于很多其他环境

应用广泛的浏览器语言 且与HTML、CSS完全集成

很多语言可以被“编译”成JavaScript

Typescript 为典型 需要掌握JavaScript 进一步了解学习
