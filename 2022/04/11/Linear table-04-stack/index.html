栈Stack

如何实现浏览器的前进和后退功能

后进者先出，先进者后出

栈是一种“操作受限”的线性表，只允许在一端插入和删除数据

从功能上来说，数组或链表确实可以替代栈

数组或链表暴露了太多的操作接口

只涉及在一端插入和删除数据 FILO

入栈和出栈

数组实现的栈，我们叫作顺序栈，用链表实现的栈，我们叫作链式栈

顺序栈

存储数据只需要一个大小为 n 的数组 只需要一两个临时变量存储空间

空间复杂度是 O(1)

只涉及栈顶个别数据的操作，所以时间复杂度都是 O(1)

栈空间不够时，我们重新申请一个是原来大小两倍的数组

先复制 之前的栈 然后压入新的

入栈操作的均摊时间复杂度就为 O(1)

栈在函数调用中的应用

函数调用要用“栈”来保存临时变量

栈在表达式求值中的应用

编译器 两个栈来实现的

  • 一个保存操作数的栈
  • 一个保存运算符的栈

  1. 遇到数字，我们就直接压入操作数栈

  2. 遇到运算符，就与运算符栈的栈顶元素进行比较

    优先级高，就将当前运算符压入栈

  3. 低或者相同

    运算符栈中取栈顶运算符、操作数栈的栈顶取 2 个操作数

  4. 计算完的结果压入操作数栈

3+5*8-6

栈在括号匹配中的应用

借助栈来检查表达式中的括号是否匹配

用栈来保存未匹配的左括号，从左到右依次扫描字符串

左括号时，则将其压入栈中；

右括号时，从栈顶取出一个左括号。

不能配对 栈中没有数据 非法

开篇

顺序查看了 a，b，c 三个页面

后退按钮 页面 c 后退到页面 a

前进按钮回到 b 页面

页面 b —-新的页面 d 注意清空了栈Y

不管基于数组还是链表，入栈、出栈的时间复杂度都为 O(1)
