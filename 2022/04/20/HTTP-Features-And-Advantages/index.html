HTTP Features And Advantages

Created2022-04-20|Updated2022-04-20
|Post View:

HTTP有哪些特点？

HTTP 协议的特点

灵活可扩展

HTTP 协议是一个“灵活可扩展”的传输协议

空格分隔单词，换行分隔字段

可靠传输

HTTP 协议是一个“可靠”的传输协议

网络基本正常的情况下数据收发必定成功

HTTP 协议是一个应用层的协议

FTP 只能传输文件、SMTP 只能发送邮件、SSH 只能远程登录

HTTP 凭借着可携带任意头字段和实体数据的报文结构，以及连接控制、缓存代理等方便易用的特性 通用的数据传输方面

请求 - 应答

HTTP 协议使用的是请求 - 应答通信模式

HTTP 请求处理封装成远程函数调用

无状态第五个特点

HTTP 协议是无状态的。

“状态”是客户端或者服务器里保存的一些数据或者标志

记录了通信过程中的一些变化信息。

传输的实体数据

可缓存可压缩 可分段获取数据 支持身份认证 支持国际化语言

HTTP 是灵活可扩展的，可以任意添加头字段实现任意功能；

HTTP 是可靠传输协议，基于 TCP/IP 协议“尽量”保证数据的送达；

HTTP 是应用层协议，比 FTP、SSH 等更通用功能更多，能够传输任意数据；

HTTP 使用了请求 - 应答模式，客户端主动发起请求，服务器被动回复请求；

HTTP 本质上是无状态的，每个请求都是互相独立、毫无关联的，协议不要求客户端或服务器记录请求相关的信息。

HTTP有哪些优点？又有哪些缺点？

HTTP 最重要也是最突出的优点是“简单、灵活、易于扩展”

下层可以随意变化，而上层的语义则始终保持稳定。

应用广泛、环境成熟

都有 HTTP 调用库和外围的开发测试工具

具有“跨语言、跨平台”的优越性

无状态

既是优点也是缺点

HTTP 协议最好是既“无状态”又“有状态” “小甜饼”Cookie 技术

明文

“双刃剑”，就是明文传输

用简单可阅读的文本形式

开发调试工作带来极大的便利

黑客就是利用了 HTTP 明文传输的缺点

不安全

“身份认证”和“完整性校验”这两方面 HTTP 也是欠缺的

解决 HTTP 不安全的缺点，所以就出现了 HTTPS

性能

不算差，不够好

终极解决方案：HTTP/2 和 HTTP/3

HTTP 最大的优点是简单、灵活和易于扩展；

HTTP 拥有成熟的软硬件环境，应用的非常广泛，是互联网的基础设施；

HTTP 是无状态的，可以轻松实现集群化，扩展性能，但有时也需要用 Cookie 技术来实现“有状态”；

HTTP 是明文传输，数据完全肉眼可见，能够方便地研究分析，但也容易被窃听；

HTTP 是不安全的，无法验证通信双方的身份，也不能判断报文是否被篡改；

HTTP 的性能不算差，但不完全适应现在的互联网，还有很大的提升空间。
