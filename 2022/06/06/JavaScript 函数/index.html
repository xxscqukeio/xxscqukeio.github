JavaScript 函数

函数本质–对象

每个函数–Function类型的实例 Function 也有属性和方法（同引用类型一样）

函数名–指向函数对象的指针（将函数名 想象为指针） 不一定与函数本身绑定

4种定义函数的方式

不推荐使用Function来定义函数 这段代码会被解释2次

1.作为常规ECMAScript代码

2.解释给构造函数字符串

解释2次 导致影响性能

1.箭头函数

使用（=>）定义函数表达式

简洁的语法适合嵌入函数

一个参数场景–可省略括号

没有参数，多个参数–使用括号

大括号{ }–包含”函数体”—多条函数语句

箭头函数–也可以省略大括号{ }—改变函数行为—-不使用{ } =>只能一行代码

省略{ } – 隐式返回 这行代码的值 不需要return

箭头函数-语法简洁，多场合不适用–不能使用arguments、super 和 new.target

—不能作为构造函数 —-没有prototype属性

2.函数名

函数名-指向函数对象的指针–包含对象指针的变量具有相同行为

—- 一个函数可以有多个名称

在ECMAScript6 - 所有函数对象会暴露只读属性name（包含函数信息）

–多数情况下，保存函数标识符（字符串化的变量名）—函数没有名称（空字符串）—- Function构造函数 （anonymous匿名的）

3.理解参数

ECAMScript函数–参数表现为数组—函数被调用（接受一个数组）

使用function关键字-定义（非箭头）函数–函数内部访问arguments对象

arguments对象-类数组对象（非array实例–arguments[0])

参数总数–arguments.length属性

参数写出–方便但非必须—不存在验证命名参数的机制

通过arguments.length属性 检查传入的参数个数

开发者传入任意参数的个数

命名参数与arguments对象 – 使用谁都无所谓

arguments[ ] 与 传入参数

如果只传入了一个参数arguments[1]设置的值–不会传入第二个命名参数

arguments对象的长度–根据传入的参数个数

命名参数–如果调用函数没传入参数—undefined—-定义了参数但未初始化

如 doAdd()传了一个参数 num2为undefined

严格模式下 函数中重写arguments[ ]导致语法报错

3.2 箭头函数中的参数

箭头函数没有arguments[ ] 但可以包装在函数中 读取外部arguments[ ]

所有函数都是 按值传递 不可能按照引用传递

对象作为参数传递 传递的值 – 这个对象的引用

4.没有重载

ECMAScript 函数 — 不能重载

例如Java中 一个函数可以有2个定义 – 只要签名（接受参数的类型、数量）不同

ECMA没有签名–没有重载 参数–零个或多个值的数组

如果先后定义了2个同名函数 后定义的会覆盖先定义的

函数名—指针—-ECMAScript没有函数重载

后定义的函数会重写定义先定义的

5.默认参数

es5前–实现默认参数—检测某个参数是否等于undefined—-没有传入则赋值

es6 支持显式定义 默认参数 – name = “henry”（参数=默认值）

可以利用多个独立的默认值

es5严格模式下 – arguments 仍然以调用函数时 传入值为准

默认参数不限于 原始类型 或者 对象类型

也可以调用函数返回值

箭头函数也可以使用默认参数

5.1默认参数作用域与暂时性死区

默认参数–会按照定义它们的顺序依次初始化

6.参数扩展与收集

es6 – 新增扩展操作符 — 简洁地操作和组合 集合数据

—-应用场景—–参数列表 ——- 利用语言弱类型及参数长度可变

普通函数和箭头函数 中 可以用于命名参数

6.2 收集参数

收集参数–得到一个Array实例

收集参数–只能作为最后一个参数

收集参数 并不影响arguments对象 — 调用时 传递函数的参数

2.2 期约基础

1.期约状态机

期约实例 – 传给console.log( ) — 控制台输出 实例处于待定（pending）

期约 – 有状态的对象 — 3种状态

待定pending、兑现fulfilled 解决resolved、拒绝rejected

待定（pending）期约最初始状态

待定状态下期约–落定settled 2种状态一种–兑现（fulfilled）或者拒绝（rejected）状态 — 落定后 状态不可逆

期约 – 甚至永远处于待定（pending）状态

期约状态 — 私有的 — 不能直接通过JavaScript检测 读取 修改

期约故意 – 将异步行为封装起来 — 隔离外部的同步代码

2.解决值、拒绝理由、期约用例

期约 2大用途

  1. 抽象地表示一个异步操作 – 期约的状态（期约是否完成）

    待定（pending）尚未开始或者正在执行中

    兑现（fulfilled）已经成功完成

    拒绝（rejected）没有成功完成

    期约–服务器http请求— 200-299状态码（兑现）不在范围（拒绝）

  2. 期约–兑现—私有内部值value

    拒绝—私有内部理由reason

    值或者理由 – 包含原始值或者对象的不可修改引用 — 可选的 –默认undefined

    对应落定状态 – 值或理由

3.通过执行函数控制期约状态

期约状态–私有化

执行函数2种职责 —

1.初始化期约的异步行为

2.控制状态的最终转换

控制期约状态转换—调用2个函数参数实现resolve（）和reject（）

调用resolve（）状态–兑现fulfilled

调用reject（）状态–拒绝 抛出错误

4.Promise.resolve( ) – 通过执行函数才能转换为落定状态

期约并非 – 一开始必须处于待定状态

调用Promise.resolve( ) 静态方法 — 实例化一个解决的期约

解决的期约的值 — Promise.resolve( ) 的第一个参数

— 可以把任何值都转换为期约

静态方法 — 传入参数为期约Promise 行为类似空包装

Promise.resolve( ) 可以说是一个幂等方法

​ 幂等性 – 保留传入期约的状态

静态方法 — 能够包装任何非期约值 — 包括错误对象 —– 转换为解决的期约

错误不能通过try/catch捕获 — 只能通过拒绝处理处理程序捕获
